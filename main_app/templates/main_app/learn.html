We can transform the word together... coalisium is a web-based collaboration tool in which anyone can create or contribute to a project by adding entries to it. Each entry is a document that describes an idea or part of an idea. In the project the entries are organized in a hierarchical structure for ideas to follow a coherent flow while at the same time allowing for various solutions to coexist.

 

 

The project structure lets ideas to be matured based on four different types of entries:

路        (R) Requirement: states the general problem and what wants to be achieved.

路        (C) Concept: description of the main idea of how the problem stated in the requirement will be solved.

路        (D) Design: complete definition of the solution proposed in the concept in such a way that it can be accurately reproduced by others.

路        (V) Validation: evaluation of the design vs the requirements through analyses or tests.


New entries are created from previous entries. For example, when a project is created it gets created with a (R) Requirement entry since this is where the scope of the project is defined. Once the first requirement is created now anyone can create a (C) Concept entry for it, when a Concept entry is created anyone can now create a (D) Design entry for it, and finally when that one is created anyone can create a (V) Validation entry for it. On the project tree structure these relationships are shown with a line linking the different entries.



Also, more than one entry can be created from another. For example a Requirement can have more than one Concept since there is more than one way to solve a problem, a Concept can have more than one Design since an idea of solution can be developed in different ways and naturally a Design can as well have more than one Validation. Anyone can create as many entries of any type at any level as he or she desires. On the project structure tree any new entry is represented as a new branch.



Furthermore, the structure permits any problem to be dissected into smaller sub-problems. This can be done by the creator of a Concept entry by attaching Requirement sub entries to it. For example the Concept for an electric boat would define the general idea of how the boat will work and the different systems that would encompass it, then the creator of the Concept could create a Requirement for each one of the sub-systems (Mechanical, Electrical, Controls, etc.) to define what each one of those sub-systems would need for the complete solution to work as a whole.
